How it works

Simple by design

Kenari sits between your users and your monitoring tools, handling authentication so your tools don't have to.

01

Deploy Kenari on the edge

Run on Cloudflare Pages — geographically separate from your infrastructure.

02

Configure your routes

Add your monitoring tools in monitor.config.ts. No UI required.

03

Login once

Email/password or GitHub OAuth. Session managed by Lucia auth.

04

Access everything

Kenari proxies requests with auth headers injected. Your tools never see unauthenticated traffic.

Features

Everything you need

Built for teams who take monitoring seriously.

🔐

Single Sign-On

One login for all your monitoring tools. Email/password and GitHub OAuth out of the box.

🌍

Edge-first

Deploy on Cloudflare Pages, Vercel, or any edge runtime. Separate from what you monitor.

🔌

Config-driven

Add monitoring routes in a single TypeScript file. No database migrations, no UI clicks.

🟢

Health checks

Real-time upstream status on your dashboard. Know immediately when something is down.

🦀

Kenari CLI coming soon

Rust agent installed on monitored hosts. Sends metrics without exposing inbound ports.

📋

Audit log v0.2

Who accessed what, when. Full visibility into your team's monitoring activity.

Architecture

Designed for resilience

Kenari lives on the edge. Your monitoring tools live on your servers. They never need to be in the same place.
